# SWOTLY — Devir & Rehber (Claude Code VEYA yeni chat oturumu için)

Mustafa ile Türkçe çalışıyoruz; kod, commit mesajları ve UI metinleri İngilizce (British English).
Yeni bir CHAT oturumunda: bu zip son çalışan sürümdür — doğrudan yamala, sıfırdan yazma, her turda tek zip üret ve sonda bu dosyanın "Bilinen durum" bölümünü güncelle. Token disiplini: toplu feedback → toplu yama → tek check → tek zip.

## Proje
Expo/React Native (SDK 54, TypeScript) iPhone uygulaması. Mustafa'nın İngilizce koç ders notlarını (PDF/foto/metin) Claude API ile analiz edip konsept + egzersizlere çevirir; SRS (SM-2) tabanlı çok günlü takvim planı üretir. Local-first: AsyncStorage (`swotly:v1`), API anahtarları SecureStore'da. Model: claude-sonnet-4-6, doğrudan fetch. Mustafa iPhone-only test eder: GitHub Codespaces + `npx expo start --tunnel` + Expo Go.

## ÇALIŞMA KURALLARI (kritik — her oturumda geçerli)
- **TOKEN DİYETİ (okuma disiplini):** Dosyayı ASLA baştan sona `view` etme. Önce `grep -n` ile hedefi bul, sonra yalnız dar satır aralığını oku (`sed -n a,bp` / view_range). Aynı dosyayı iki kez okuma — gerekirse ilk okumada geniş tut. SessionScreen (1100+ satır) özellikle: sadece ilgili blok. Doğrulama için tam dosya okuma yerine `scripts/check.sh` yeterli. Keşif = tek birleşik grep komutu (çok desen tek çağrıda, `&&` ile zincirle).
1. **Yamala, sıfırdan yazma.** Mevcut dosyaları cerrahi düzenle; çalışan kodu yeniden üretme.
2. **Önce mevcut olanı ara.** Yeni bir öğe/özellik eklemeden önce aynı işlevi karşılayan bir şey var mı diye TÜM ekranları tara (ör. geçmişte 3 ayrı "Resume" öğesi birikmişti). Çakışma varsa ekleme — mevcut olanı iyileştir veya kaldırılmasını öner.
3. **Her anlamlı değişiklikten sonra doğrula:** `bash scripts/check.sh` (dosya başına tsc + import/export guard). Bilinen ön-mevcut false-positive'ler: TS2591 (require, UI.tsx), TS2339 (.at/.prev, HomeScreen), TS2741 children, TS7006/TS7016/TS18046/TS7053/TS7031. Bunların DIŞINDA yeni hata bırakma.
4. **Her özellik sonrası `git commit`** — küçük, geri alınabilir adımlar. Mesaj İngilizce, tek satır.
5. **Migration disiplini:** AppData şeması değişirse `storage.ts loadData` içinde bayraklı tek-seferlik migration ekle (örnekler: `imgV3`, `oddFixV1`) ve `types.ts emptyData`'ya bayrağı koy.
6. **Resume güvenliği:** Seans pozisyonu cevaplanmış soru ID'lerinden türetilir (indeks değil). Yeni step tipi eklersen `initialIdx` döngüsüne resume kuralını da ekle; rastgelelik deterministik olmalı (`hash()` ile).
7. **Rapor formatı:** Değişiklik sonrası Mustafa'ya madde-madde Türkçe özet + kök-neden açıklaması + kısa test rotası ver.

## Mimari kararlar
- **PDF:** cihazda pdf-lib ile 8'er sayfa dilimlenir → paralel 3 analiz, kompakt-retry, `salvageJson` parantez-dengeleyici onarıcı. Story alanı chunk birleştirmelerinde taşınmalı (v0.24'te düzeltildi — bozma).
- **Görseller (3 katman):** ① ConceptVisual (kategori-degrade+emoji, anlık) ② arka plan hattı `localizeImagesInBackground`: Openverse(+Wikipedia) adayları → Claude süzgeci (keskin tur + %25 kotaya kadar gevşek tur) → `netq.ts` nazik kuyruğuyla diske indirme + format doğrulama; kart `file://` olunca fotoğrafa yükselir ③ opsiyonel Pexels key. UI ASLA uzak URL render etmez. Pollinations katmanları uykuda (bakiye 0). Teşhis: Settings → Advanced → "Run picture fetch now" (targets/cands/verified/saved/err).
- **Ses:** Google Translate TTS indir-önbelleğe-çal (gb/us/device; `_` "blank" okunmasın diye temizlenir), cihaz sesi yedek. Otomatik seslendirmeler `say()` üzerinden geçer (mute'a saygı); elle dokunulan hoparlörler her zaman çalar. UI ses efektleri `src/sfx.ts`: WAV'lar çalışma anında sentezlenip cache'e yazılır (asset/bağımlılık YOK, expo-audio zaten mevcut); ok/bad/tick/fanfare; Session'da `sfx()` yardımcısı mute'a saygılı; ses değiştirirsen sfx.ts'teki V sabitini artır (cache yenilensin).
- **Plan:** buildStudyPlan kategori round-robin, konsept bölünmez; gün seansı = günün TAMAMI + vadeli tekrarlar; Home hedefi YALNIZCA bugünün plan gününü yansıtır. Günlük session log BİRİKİR (üzerine yazılmaz — v0.23 bug fix, bozma).
- **Soru motoru:** tanıma→üretim sırası + kavramlar arası staggered round-robin; Mystery ribbon (cevap sırasında konsept adı gizli); şıksız mcq/odd/stress ASLA type'a düşürülmez — atılır (import) / filtrelenir (session); odd/mcq şıkları kayıtta saklanır ve cevap şıklar arasında garanti; gap/type cümlesi cevabı içeremez; hint cevabın kelimesini içeremez; domain rotasyonu zorunlu ("VARIETY IS SACRED" + reply-genelinde senaryo/isim/yer tekliği); import'ta near-twin filtresi (aynı konseptte kelime-örtüşmesi ≥0.6, aynı import'ta iskelet/0.7 örtüşme → at). Speed Round finali (5 soru, 8sn görünür geri sayım, +2XP, SRS'e işlemez, exId "spd:"); Story Cloze (exId "sto:", kısmi +2XP/boşluk); fix tipi (hata düzeltme); stress tipi (Pronunciation: heceler choices'ta SÖYLENİŞ SIRASINDA, cevap = vurgulu hece, listede TEK geçmeli — karıştırma/dedupe YAPMA). Duel tipi DENENDİ ve KALDIRILDI (speed round ile çakışıyordu) — geri getirme. Pronunciation self-grade "echo/say it out loud" kutusu v0.31'de KALDIRILDI (gap sorularını rehin alıyordu, objektif ölçüm yoktu) — geri getirme; Pronunciation yeni reçetesi: stress + sound-match mcq + listen.
- **XP:** results reduce formülü tek gerçek kaynak (pic:5, spd:2, sto:0 — story speedXpRef'ten kısmi öder). Aynı XP'yi iki yerden ödeme (v0.23'te double-count düzeltildi).

- Python yamalarında geniş regex YASAK: dar, assert'li, birebir string eşleşmesi kullan (bir kez buildStudyPlan'ın kuyruğu silindi).
